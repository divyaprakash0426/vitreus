Vitreus: Local-First Agentic Spreadsheet Intelligence1. Project SummaryVitreus is a local-first AI agent designed to bridge the gap between complex spreadsheet data and intuitive natural language. By utilizing Gemma 4’s massive 256K context window (31B) and native multimodal capabilities, Vitreus allows users to "talk" to their LibreOffice Calc sheets in real-time. It extracts raw data via the PyUNO bridge and uses vision to interpret charts and physical receipts, all while ensuring data privacy by running entirely on local hardware (via Ollama) or secure enterprise endpoints (Google AI Studio).

3. Tech Stack & LibrariesInference Engine: Ollama (Local) with Gemma 4 31B Dense and 4B Assistant drafter.Cloud API: Google AI Studio (Gemini API for Gemma 4) for high-scale reasoning.Spreadsheet Bridge: PyUNO (Native LibreOffice) + unotools (Abstraction layer).Agent Framework: PydanticAI for structured tool-calling and schema validation.Vision: Pillow for image preprocessing before feeding to Gemma 4's multimodal encoder.

5. Implementation Roadmap
Phase 1: The Invisible Hand. Establish a stable PyUNO connection to a visible Calc window and execute basic "Cell Highlighting."
Phase 2: The Deep Context. Implement a "Data-to-Prompt" pipeline that fits entire workbooks into Gemma 4's context window.
Phase 3: Visual Witness. Integrate multimodal input to explain live charts or import data from receipt photos.
Phase 4: Shell Sync. Finalize the calc.nu script to allow terminal users to pipe shell results directly into formatted spreadsheet tables.
